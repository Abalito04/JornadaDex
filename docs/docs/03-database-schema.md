# Database Schema

## Tables

### companies

| Column | Type | Notes |
| --- | --- | --- |
| id | Integer | Primary key |
| name | String(150) | Required, unique |
| tax_id | String(50) | Optional |
| active | Boolean | Default true |
| created_at | DateTime | Required |
| updated_at | DateTime | Required |
| deleted_at | DateTime | Soft delete |
| created_by | Integer | FK users.id |
| updated_by | Integer | FK users.id |
| deleted_by | Integer | FK users.id |

### branches

| Column | Type | Notes |
| --- | --- | --- |
| id | Integer | Primary key |
| company_id | Integer | FK companies.id |
| name | String(150) | Required |
| address | String(255) | Optional |
| active | Boolean | Default true |
| audit fields | mixed | Standard audit columns |

### departments

| Column | Type | Notes |
| --- | --- | --- |
| id | Integer | Primary key |
| company_id | Integer | FK companies.id |
| name | String(150) | Required |
| active | Boolean | Default true |
| audit fields | mixed | Standard audit columns |

### users

| Column | Type | Notes |
| --- | --- | --- |
| id | Integer | Primary key |
| company_id | Integer | FK companies.id |
| username | String(80) | Required, unique |
| email | String(255) | Required, unique |
| password_hash | String(255) | Required |
| role | String(30) | Administrator, Supervisor, Employee |
| is_company_owner | Boolean | True for the company owner / jefe user |
| is_active | Boolean | Default true |
| last_login_at | DateTime | Optional |
| audit fields | mixed | Standard audit columns |

Rules:
- A normal user belongs to exactly one company.
- A company must have at least one active owner user.
- Employee users must be linked to an employee in the same company.
- Users cannot access records from another company.

| description | Text | Optional |

### time_records

| Column | Type | Notes |
| --- | --- | --- |
| id | Integer | Primary key |
| company_id | Integer | FK companies.id, denormalized for tenant filtering |
| employee_id | Integer | FK employees.id |
| area_id | Integer | FK areas.id |
| task_id | Integer | FK tasks.id |
| record_date | Date | Required |
| start_time | Time | Required |
| end_time | Time | Nullable until task is finished |
| hours | Numeric(8, 2) | Required, calculated, default 0.00 while in progress |
| observations | Text | Optional |
| audit fields | mixed | Standard audit columns |

Constraints:
- start and end timestamps are generated by the server
- only one active task without `end_time` is allowed per employee
- prevent exact duplicate employee/date/start/end/task active entries
- task must belong to selected area
- employee, area, task and time record must belong to the same company
